# Grounding and Retrieval Pattern

## Problem

Ungrounded answers reduce trust and create factual errors.

## When to Use

Use when responses depend on enterprise records or curated knowledge.

## When Not to Use

Do not use when task can be solved deterministically without retrieval.

## Tradeoffs

Requires indexing pipelines and metadata governance.
